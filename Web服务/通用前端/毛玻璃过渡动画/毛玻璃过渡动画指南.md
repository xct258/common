# 毛玻璃（backdrop-filter）过渡动画指南

## 问题

`backdrop-filter: blur()` 在父元素 `opacity < 1` 时无法正常工作。

**现象**：父元素做 `opacity: 0 → 1` 过渡时，子元素的 `backdrop-filter` 在过渡期间完全不渲染模糊效果，直到父元素 `opacity` 达到 `1` 的瞬间才突然出现，产生闪烁。

**原因**：CSS 规范要求 `opacity < 1` 的元素创建新的合成层（compositing group）。`backdrop-filter` 在该合成层内只能模糊层内内容，而非真正的背景。当 `opacity` 到达 `1` 时合成层消失，`backdrop-filter` 突然切换到模糊真实背景。

## 解决方案

**在目标元素自身上用 `@keyframes` 动画同时过渡 `opacity`、`backdrop-filter` 和 `background`，不要在父元素上做 `opacity` 过渡。**

### 基本用法

```css
/* 初始状态：隐藏 */
.glass-card {
  opacity: 0;
}

/* 触发动画的 class */
.glass-card.show {
  animation: glass-reveal 0.8s ease both;
}

/* 同步过渡 opacity + backdrop-filter + background */
@keyframes glass-reveal {
  from {
    opacity: 0;
    backdrop-filter: blur(0px);
    background: transparent;
  }
  to {
    opacity: 1;
    backdrop-filter: blur(16px);
    background: rgba(255, 255, 255, 0.15);
  }
}
```

```js
// JS 触发
element.classList.add('show');
```

### 配合 background-switcher.js 使用

JS 会在页面加载完成后给 `<html>` 添加 `bg-switcher-page-revealed` class。

```css
/* 加载期间隐藏内容 */
html.bg-switcher-page-hidden .card {
  opacity: 0;
}

/* 加载完成后触发动画 */
html.bg-switcher-page-revealed .card {
  animation: glass-reveal 0.8s ease both;
}

@keyframes glass-reveal {
  from {
    opacity: 0;
    backdrop-filter: blur(0px);
    background: transparent;
  }
  to {
    opacity: 1;
    backdrop-filter: blur(16px);
    background: rgba(255, 255, 255, 0.15);
  }
}
```

### 多元素交错入场

```css
.card:nth-child(1) { animation-delay: 0s; }
.card:nth-child(2) { animation-delay: 0.15s; }
.card:nth-child(3) { animation-delay: 0.3s; }
```

## 错误做法

```css
/* ❌ 不要这样做：父元素 opacity 过渡会破坏子元素的 backdrop-filter */
body {
  opacity: 0;
  transition: opacity 0.8s ease;
}
body.visible {
  opacity: 1;
}

.card {
  backdrop-filter: blur(16px); /* 在 body opacity < 1 期间不会正常渲染 */
}
```

```css
/* ❌ 也不要用 CSS transition 过渡 backdrop-filter（浏览器支持差） */
.card {
  backdrop-filter: blur(0px);
  transition: backdrop-filter 0.8s ease;
}
.card.show {
  backdrop-filter: blur(16px);
}
```

## 规则总结

| 做法 | 结果 |
|------|------|
| 父元素 `opacity` 过渡 + 子元素 `backdrop-filter` | ❌ 模糊在过渡结束后突然出现 |
| `transition: backdrop-filter` | ❌ 浏览器兼容性差，部分不生效 |
| `@keyframes` 动画在目标元素上同步过渡 | ✅ 平滑效果 |
